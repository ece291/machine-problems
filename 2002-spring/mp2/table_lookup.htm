Table Lookup

Using lookup tables is a good way to organize your data, and in some situations can make your code more efficient by eliminating conditional jumps.  Look over lookup tables in lecture eight.  In this MP, the Dispatch function will use the movement array to get the inputs for either string matching algorithm.  The entries in the movement array correspond to how much you have to offset by to traverse in a particular direction in the word grid.  For instance, the first entry in movement corresponds to traversal in the upperleft direction.  In the word grid this value would be one row's length minus one entries size.  This is -(NUM_COLS+2)-1 = -3 - NUM_COLS.  Also remember to take into account the fact that the entries of the movement array are word sized, not byte sized.

When writing Dispatch, be sure to come up with your game plan before starting.  Dispatch can be written with just two medium sized loops.  Remember that you only need to call the correct string matching algorithm on the horizontal and vertical border characters.  You will need to specify more than one direction from each border character.  But it would be wasteful to specify all eight directions.

;use this lookup table to figure out the offset needed to traverse the
;table in any of the eight directions
movement dw  -3 - NUM_COLS, -2 - NUM_COLS, -1 - NUM_COLS
                dw  -1  ,  0 ,  1
                dw   1 + NUM_COLS,  2 + NUM_COLS,  3 + NUM_COLS

;the values of the movement table correspond to these directions:

movement dw     upleft,     up,     upright
                dw      left  , 0 ,  right
                dw      downleft,  down,  downright
