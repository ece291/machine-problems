Protected Mode Grading Sheet

You have to do these questions as your going along in the tutorial.  Otherwise they'll be very out of context.  As you go along, jot down some answers to these questions.  When you "handin," the TA will will go over all of these questions with you and make sure you know everything.  Each question you know will be worth about a third of a point for a total of five points for the questions.  The point is not to trick you, but to make sure you understand what's going on.

Question1: How running the goal make you feel?  Scared?  Intimidated?  Excited to know you'll learn cool stuff?  Pissed that you're going to be forced to do all this crap?  Happy that it doesn't look like there's a Parse function involved?
 

 
Question2: What is the opcode for ret?  How many bytes were the two opcodes?
 

 
Question3: How much of the mov opcode was actually opcode and how much data?
 

 
Question4: How does NASM treat labels with a dot before them to keep those local names form interfering with each other?  It renames them in order to keep all the names unique.  How does it rename them?
 

 
Question5: How would you write and set a protected mode callback or handler?  Give some key numbers.
 

 
Question6: How does the "size" column relate to the size you asked for?  The data stored in the LDT which gets displayed in the second column isn't really the size.  What is it?  (Think offsets.)
 

 
Question7: What happened when you didn't allocate enough memory in your buffer?  How was the memory was protected?
 

 
Question8: Why do you have to do both interrupts?
 

 
Question9: What did the 320x200x256 color LDT entry look like?  Give numbers.
 

 
Question10: From the math Pete does with EBP and ESP, which way does a protected mode stack grow, and which way does it shrink?  Compare it to the Real Mode stack.
 

 
Question11: What interrupts are Pete's routines calling?  What do they do?  Give me numbers and names.
 

 
Question12: How would you set the Set VESA Video Mode to 1024*768*24bit color so that we could access the screen using a linear array?  Give me numbers!
 

 
Question13: What interrupt and subfunction does _AllocMem call to actually allocate the memory?  Look them up in the DPMI Functions by Functional Group and write down names.  What does the description of this interrupt say about DPMI hosts that support virtual memory?
 

 

 
Question14: What interrupts and subfunctions does _LockMem call?  There are quite a few here.
