How to Do this MP

I could have a huge reference section, and then a huge code description section, but I decided to do a little bit at a time in a tutorial format.  "Here's how this works.  Now here's how you write the code.  Now write the code."  It's probably not worth reading over the whole thing before you start.  Just sit down at a computer and go through it step by step.  It's going to some time to go through this, but you're going to learn everything the right way with no "black boxes" or mysterious files given to you by the TAs.  If a file does seem mysterious, type it in instead of blindly pasting it in.  Look at the references.  Ask the guy (or gal) next to you.  Hopefully you'll learn not only 32-bit protected mode, but a little about how assemblers, compilers, and linkers work, learn a little about Makefiles, learn a little about graphics, and maybe even learn a little bit about yourself.  So lets begin where every good tutorial should begin -- at the end.

The Goal

The Goal is to make a cool graphics program in protected mode using a high-resolution, high-color graphics mode, which you'll play with in a second.  I want to make this the best protected-mode tutorial ever, so keep a text file of running comments.  Even if you figure it out in the next paragraph, perhaps it should have been moved up.  Send that text file to me at the end. If you have a question or if something is unclear, incomplete, or just plain wrong, e-mail me at gallicch@uiuc.edu with "MP5 -- you suck because xxxxx" as the subject.

Protected Mode and Your Final Project

Advantages

  • Better memory management
    • Can allocate huge buffers for huge pictures
    • Don't have to deal with stupid segment limitations
  • Better instructions
    • Take full advantage of advanced instructions
    • Reference memory with any register or combination
  • High color, high resolution graphics
    • Your projects will look much nicer
    • Palettes used to be a huge pain.
  • Better tools
    • NASM is much cleaner and more consistent MASM
    • NSAM uses standard formats and is compatible with standard GNU tools.
  • Pete's Library
    • We give you libraries with source code for some pretty cool stuff.

Disadvantages

  • You're not as used to it's beauty.
  • Doing hardware stuff will be harder.
  • Doing sound will be harder
  • Doing networking will be harder
  • Debugging will be harder

If you use Protected Mode

  • If you do sound in your project, make it a nice Pete-like library so it's no longer harder.
  • If you do networking in your project, it it a nice Pete-like library so it's no longer harder.
  • Document your successes and tribulations for others in the future
  • If you contribute to the future of protected mode programming in the class, you will be more kindly graded.
